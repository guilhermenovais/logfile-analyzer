# Data Model: Tauri Auto-Updater

## Entities

### Update Manifest (`latest.json`)

Generated by `tauri-action` during CI builds and uploaded to GitHub Releases.

| Field | Type | Description |
|-------|------|-------------|
| `version` | `string` | Semantic version of the latest release (e.g., `"0.2.0"`) |
| `notes` | `string` | Release notes (auto-generated from GitHub release notes) |
| `pub_date` | `string` | ISO 8601 publication timestamp |
| `platforms` | `Record<string, PlatformUpdate>` | Per-platform update info, keyed by `"{os}-{arch}"` |

#### PlatformUpdate

| Field | Type | Description |
|-------|------|-------------|
| `signature` | `string` | Base64-encoded Ed25519 signature of the update artifact |
| `url` | `string` | Download URL for the update artifact on GitHub Releases |

**Platform keys produced by the release workflow**:
- `linux-x86_64` (AppImage)
- `darwin-aarch64` and `darwin-x86_64` (universal binary via `app.tar.gz`)
- `windows-x86_64` (NSIS `.exe`)

### Signing Key Pair

| Component | Storage | Description |
|-----------|---------|-------------|
| Public key | `tauri.conf.json` → `plugins.updater.pubkey` | Embedded in the app binary; used to verify update signatures at install time |
| Private key | GitHub Actions secret `TAURI_SIGNING_PRIVATE_KEY` | Used during CI builds to sign update artifacts; never committed to the repository |
| Password | GitHub Actions secret `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | Optional passphrase for the private key (empty string if none) |

### Update Check Result (frontend)

Returned by `check()` from `@tauri-apps/plugin-updater`.

| Field | Type | Description |
|-------|------|-------------|
| `version` | `string` | Version number of the available update |
| `date` | `string \| null` | Publication date |
| `body` | `string \| null` | Release notes body |
| `downloadAndInstall` | `(onProgress?) => Promise<void>` | Method to download and install the update |

### Download Progress Event (frontend)

Passed to the `downloadAndInstall` callback.

| Event | Data | Description |
|-------|------|-------------|
| `Started` | `{ contentLength: number \| null }` | Download has begun; total size if known |
| `Progress` | `{ chunkLength: number }` | A chunk has been received |
| `Finished` | `{}` | Download complete |

## State Transitions

```
App Launch
  → Check for updates (GET latest.json)
  → [No update / Offline / Error] → Normal operation (silent)
  → [Update available] → Show notification modal
      → [User dismisses] → Normal operation
      → [User accepts] → Download with progress
          → [Download fails] → Show error, resume normal operation
          → [Signature invalid] → Show verification error, resume normal operation
          → [Download complete] → Prompt restart
              → [User restarts] → relaunch() → App starts at new version
              → [User defers] → Normal operation (check again on next launch)
```

## Relationships

- `tauri.conf.json` references the public key and the endpoint URL
- The CI workflow uses the private key secret to sign artifacts during `tauri build`
- `tauri-action` reads `tauri.conf.json` to know that `createUpdaterArtifacts` is enabled
- `tauri-action` generates `latest.json` and `.sig` files, attaching them to the GitHub Release
- The frontend `check()` call triggers the Rust plugin to fetch `latest.json` from the endpoint
- The Rust plugin verifies downloaded artifacts against the embedded public key
