# Quickstart: Rewrite Search-to-Scroll Navigation

## Prerequisites

- Rust stable (pinned via `rust-toolchain.toml`)
- Node.js / pnpm (check `package.json` `engines` field)
- Tauri v2 CLI (`cargo install tauri-cli --version '^2'`)

## Development

```bash
# Install frontend dependencies
pnpm install

# Run dev mode (frontend + Rust backend)
pnpm tauri dev
```

## Quality Gates (Constitution Principle IV)

All must pass before a task is complete:

```bash
# Frontend
npx tsc --noEmit
npx eslint .
npx vitest run

# Backend
cargo clippy -- -D warnings
cargo fmt --check
cargo test
```

## Key Files

### Backend (Rust)
- `src-tauri/src/commands/viewing.rs` — add `resolve_view_row` command
- `src-tauri/src/state.rs` — `FileRuntime.view_filter` (read-only)
- `src-tauri/src/lib.rs` — register new command in Tauri builder

### Frontend (TypeScript/React)
- `src/ipc/viewing.ts` — add `resolveViewRow` IPC wrapper
- `src/hooks/useScrollToLine.ts` — new hook (core rewrite)
- `src/components/LogViewer.tsx` — consume `useScrollToLine`, remove old effect
- `src/bindings/index.ts` — auto-generated by specta (re-run after Rust changes)

### Tests
- `src-tauri/src/commands/viewing.rs` — unit test for `resolve_view_row`
  (inline `#[cfg(test)]` module)
- `src/hooks/useScrollToLine.test.ts` — hook tests with mocked IPC
- `src/components/LogViewer.test.tsx` — update existing tests for new scroll path

## Generating Bindings

After adding or modifying a Tauri command, run:

```bash
pnpm tauri dev
```

Specta auto-generates `src/bindings/index.ts` on startup. The new
`resolveViewRow` command will appear in `commands` after the first dev build.

## Testing the Fix

1. Open a file with 100,000+ lines
2. Search for a term with matches spread throughout
3. Click a result far from the current viewport — it should scroll to the correct
   line, centered
4. Enable a time-range filter, repeat — should still scroll correctly
5. Enable line wrapping, repeat — should still scroll correctly
6. Rapidly click different results — the last click's target should be displayed
