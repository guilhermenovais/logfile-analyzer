## Summary

The search-result click-to-scroll feature fails when the target line is far from
the current viewport, especially under time-range filters. The root cause is a
file-line-index-to-virtualizer-index mapping error in `LogViewer.tsx` and the
lack of a correction pass for variable row heights (line wrapping). The fix adds
a backend `resolve_view_row` IPC command for correct index translation and
rewrites the frontend scroll mechanism into a dedicated `useScrollToLine` hook
with a two-pass scroll-and-verify approach.
